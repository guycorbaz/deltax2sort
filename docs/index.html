Sorting small parts with a delta robot

deltax2sort watches a conveyor belt through a camera, spots each part — LEGO bricks first, screws and other small objects next — and drives a Delta X2 robot to pick it off the moving belt. A hardened Rust control application with a touch-first operator interface.

Why it's interesting

Hardened where it matters

Driving real motors from software means the boring parts have to be bulletproof. The architecture puts safety ahead of features.

🛑

Preemptive emergency stop

The E-stop owns its own cloned serial ports and fires synchronously from the UI callback — bypassing every queue, mutex and pending command. Robot halt M112, conveyor halt M5.

📐

Workspace enforced in the driver

Configuration is validated at startup and workspace limits are checked inside move_to itself — an out-of-bounds target errors before a single byte of G-code leaves the machine.

✅

Motion you can trust

Every command blocks until the robot echoes physical completion (FEEDBACK id), with EOF detection and a hard 30-second deadline. On failure the orchestrator clears its queue and pauses.

👁️

OpenCV vision

Blob detection with configurable thresholds and an affine pixel→robot calibration including rotation — always based on the capture resolution the camera actually grants.

🧪

Full mock mode

cargo run -- --mock runs the complete application with simulated robot, conveyor and camera. Every hardware feature exists behind traits with real and mock implementations.

👆

Touch-first operator UI

A Slint interface designed for the Raspberry Pi 7" Touch Display (800×480): finger-sized controls, no scrolling, and the E-stop always the largest target on screen.
